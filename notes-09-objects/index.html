Objects in Javascript

Objects are essential to programming an OOP based app. Objects allow developers to "encapsulate" data inside a container. This container is often the object itself. Objects can be reused, modified or acted upon. They can be considered as mini-programs inside of a larger program.

These objects are typically tasked with performing their own specific actions and duties by using object-specific variables and functions. Except, within an object a variable is called a property and a function is called a method. These components come together to make up the pieces and parts of an object.

Accessing Object Properties

Variables inside of Objects work exactly the same way as local variables inside of functions. Accessing an object property is helpful when you need to act upon it outside of the Object.

The "this" Keyword

In OOP languages the "this" keyword is often used to reference the object without using its name. This is to allow programmers to write objects that share the same name as the parent object. The "this" keyword also allows your function/method to reference the object that it currently resides in.

Anytime, you need to reference a property or another method while you are inside of an object, you should use the "this" keyword.
